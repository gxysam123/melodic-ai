# 🔧 后端启动问题已修复

## ❌ 原问题

桌面应用启动时显示错误：
```
启动失败
无法启动后端服务器: spawn ENOTDIR
```

---

## 🔍 问题原因

**路径配置错误**：
- 开发环境中，Python 文件在项目根目录
- 打包后的应用中，Python 文件在 `resources/app/` 目录
- electron-main.cjs 使用了错误的路径

---

## ✅ 解决方案

### 修改内容

**electron-main.cjs** 第 13-18 行：

```javascript
// 修复前
const BACKEND_SCRIPT = path.join(__dirname, 'backend_server.py');

// 修复后
const isDev = process.env.NODE_ENV === 'development';
const BACKEND_SCRIPT = isDev
  ? path.join(__dirname, 'backend_server.py')
  : path.join(process.resourcesPath, 'app', 'backend_server.py');
```

**electron-main.cjs** 第 44-60 行：

```javascript
// 新增路径检查和正确的工作目录
const workDir = isDev
  ? __dirname
  : path.join(process.resourcesPath, 'app');

// 检查 Python 脚本是否存在
if (!fs.existsSync(BACKEND_SCRIPT)) {
  reject(new Error(`找不到后端脚本: ${BACKEND_SCRIPT}`));
  return;
}

// 使用正确的工作目录启动
backendProcess = spawn('python3', [BACKEND_SCRIPT], {
  cwd: workDir,
  env: { ...process.env },
});
```

## 💡 技术说明

### 打包后的应用结构

```
妙音AI.app/
└── Contents/
    ├── MacOS/
    │   └── 妙音AI              # 可执行文件
    ├── Resources/
    │   ├── app/
    │   │   ├── backend_server.py     # Python 后端
    │   │   ├── audio_processor.py    # 音频处理
    │   │   └── requirements.txt      # Python 依赖
    │   └── app.asar            # 打包的前端代码
    └── Info.plist
```

### 路径解析

- **开发环境**：`__dirname` 指向项目根目录
- **生产环境**：`process.resourcesPath` 指向 `Contents/Resources/`
- **Python 文件**：在 `Contents/Resources/app/` 目录
