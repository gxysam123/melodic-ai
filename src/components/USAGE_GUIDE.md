## 📝 完整使用流程

### 1. 上传音频文件

**方式一: 拖拽上传**
- 将音频文件拖到上传区域
- 松开鼠标即可上传

**方式二: 点击上传**
- 点击"本地文件"按钮
- 选择音频文件
- 确认上传

**支持格式:** MP3, WAV, FLAC, OGG, M4A, MP4

### 2. 播放源文件

- 上传成功后，文件信息会显示在播放器区域
- 点击 **播放按钮** (▶) 可以试听原始文件
- 点击 **暂停按钮** (⏸) 暂停播放
- 点击 **停止按钮** (⏹) 停止并重置

### 3. 选择处理模式

在左侧边栏选择:
- **🎵 智能降噪** - 去除背景噪音
- **🎤 重叠人声分离** - 分离主唱和伴奏
- **🎸 多音轨分离** - 分离所有乐器音轨

### 4. 调整参数

每个模式都有独立的参数面板：

**智能降噪参数:**
- AI 降噪强度 (0-100%)
- 保留自然混响 (开/关)
- 人声特征补偿 (开/关)

**人声分离参数:**
- 灵敏度 (0-100%)
- 聚焦模式 (主唱/伴奏/原始混音)
- 目标增强 (开/关)

**多音轨分离参数:**
- 人声音量 (0-100%)
- 鼓组音量 (0-100%)
- 贝斯音量 (0-100%)
- 钢琴音量 (0-100%)
- 其他乐器音量 (0-100%)

### 5. 开始处理

- 调整好参数后，点击底部的 **"开始处理"** 按钮
- 系统日志会实时显示处理进度
- 进度条显示处理百分比

**处理时间参考:**
- 智能降噪: 2-5秒
- 人声分离: 5-15秒  
- 多音轨分离: 10-30秒

### 6. 播放处理结果

处理完成后：
- 会自动显示 **"处理完成 - 结果预览"** 区域
- 内置音频播放器，可直接试听
- 播放器支持进度拖动、音量调节

### 7. 下载结果

- 点击 **"下载处理结果"** 按钮
- 浏览器会自动下载处理后的音频文件
- 文件格式: WAV (无损)

### 8. 重新处理

如需调整参数重新处理：
- 点击 **"重置处理"** 按钮
- 调整参数
- 再次点击 **"开始处理"**

### 前端改动

- `processFile` 函数传递 `rawFile` 参数给父组件

**文件: `src/lib/apiClient.ts`**
- 完整的 REST API 客户端
- 支持文件上传、处理、下载

### 后端改动

**文件: `backend_server.py`**
- Flask API 服务器 (端口 5001)
- 文件上传接口 `/api/upload`
- 降噪处理接口 `/api/process/denoise`
- 人声分离接口 `/api/process/vocal-separation`
- 多音轨分离接口 `/api/process/track-separation`
- 文件下载接口 `/api/download/<task_id>`

## 🐛 故障排除

### 问题 1: 上传失败
**症状:** 点击上传后无响应  
**解决:**
1. 检查后端是否运行 (`curl http://localhost:5001/api/health`)
2. 检查浏览器控制台错误
3. 检查文件大小是否超过 500MB

### 问题 2: 无法播放源文件
**症状:** 点击播放按钮无反应  
**解决:**
1. 检查浏览器是否支持音频格式
2. 查看浏览器控制台是否有 CORS 错误
3. 确认文件已成功上传

### 问题 3: 处理一直卡住
**症状:** 进度条不动或一直显示"正在处理"  
**解决:**
1. 查看后端日志 (`backend.log`)
2. 检查后端是否有错误输出
3. 确认 Demucs 模型已下载完成

### 问题 4: 无法下载结果
**症状:** 点击下载按钮无反应  
**解决:**
1. 检查 task_id 是否有效
2. 确认处理已完成 (status = 'completed')
3. 检查后端输出目录是否有文件
